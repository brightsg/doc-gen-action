You are generating architecture documentation for developers joining this project. Explain how the system is structured and why.

## Required Sections

1. **System Overview** — what this system does, where it fits in the broader platform
2. **Project Structure** — each project with its responsibility, organised by layer
3. **Dependency Flow** — mermaid diagram showing project dependencies
4. **Key Design Patterns** — patterns used (repository, mediator, etc.) with brief explanation of how they're applied
5. **Data Flow** — for the 2-3 most important use cases, describe the flow from entry point to completion
6. **Infrastructure** — databases, message brokers, external services, how they connect

## Rules

- Use mermaid diagrams for dependency flow and data flow
- Focus on the "why" behind architectural decisions, not just the "what"
- Reference actual class and interface names
- Keep it high-level enough to be useful as an introduction
